# git commit -m "emoji"

Emojis are a simple and fast way to communicate something.

A commit must clearly explain the project's changes.

Why not combine these two?

---

### Table of emojis

|Emoji|Commit|
|:-:|:-|
|🎉| Project beginning |
|✨| New feature |
|🚧| Feature in progress |
|✅| Finished feature |
|🐛| Bug fix |
|🚀| Deploy stuff |
|🏷️| Release |
|⚡️| General update |
|🛠️| Maintenance |
|♻️| Refactoring |
|⚙️| Functionality |
|🎨| Style |
|🔍️| SEO |
|📦| Package |
|📂| Directory |
|📑| Documentation |
|⏪️| Revert changes |
|⚠️| Problems in the code |
|💩| Temporary bad code |

### Attention!

Emojis are not enough to explain a change, a **great description** must be part of the commit. 

`Have fun!`
